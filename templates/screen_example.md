# 화면 기획서: AI 초안 생성 결과

> 작성일: 2026-02-23 | 작성자: 기획팀 | 버전: v1.0

---

## 1. 사용자 목표 (User Goal)

- AI가 만들어 준 초안을 확인하고, 마음에 들면 편집기로 넘어가고 싶다.
- 마음에 들지 않으면 조건을 바꿔서 다시 생성하고 싶다.

## 2. 시스템 목표 (System Goal)

- 사용자가 초안을 수락하여 편집 단계로 진입하게 한다.
- 재생성 시에도 이탈하지 않고 서비스 안에서 반복 시도하게 한다.
- 성공 지표: 초안 수락률 60% 이상

## 3. 데이터 모델 (Data Model)

### 표시 데이터 (이 화면이 보여주는 것)

| 데이터 항목    | 타입                         | 설명/제약                    | 출처                  |
| -------------- | ---------------------------- | ---------------------------- | --------------------- |
| 생성된 초안 본문 | 텍스트                     | 최소 200자, 최대 3,000자     | AI 생성 API 응답      |
| 초안 제목      | 텍스트                       | 최대 60자                    | AI 생성 API 응답      |
| 사용된 톤      | 선택지(격식체, 친근체, 전문가) | 선택지 중 1개              | 이전 화면에서 전달    |
| 사용된 주제    | 텍스트                       | 사용자가 입력한 원본 주제    | 이전 화면에서 전달    |
| 글자 수        | 숫자                         | 생성된 본문의 글자 수        | 프론트에서 계산       |
| 생성 소요 시간 | 숫자                         | 초 단위, 소수점 1자리        | AI 생성 API 응답      |

### 데이터 흐름

```
[주제 입력 화면] →(주제, 톤, 길이 옵션)→ [이 화면: 초안 결과] →(초안 본문, 제목)→ [편집기 화면]
                                                    ↺ 재생성 시 (수정 지시사항 포함) 자기 자신으로 루프
```

## 4. 정보 위계 (Information Hierarchy)

### Primary (핵심 정보) — 스크롤 없이 즉시 인지

1. **생성된 초안 본문** — 사용자가 이 화면에 온 유일한 이유. 가장 넓은 영역을 차지해야 함
2. **초안 제목** — 본문 위에 위치하여 글의 방향을 즉시 파악하게 함

### Secondary (보조 정보) — 맥락 보완

- 사용된 주제/톤 태그 (상단, 항상 표시) — "내가 요청한 조건이 맞는지" 확인용
- 글자 수 (본문 하단, 항상 표시)
- 생성 소요 시간 (본문 하단, 항상 표시) — 서비스 속도에 대한 신뢰감
- "AI가 생성한 초안입니다. 편집기에서 자유롭게 수정할 수 있어요." 안내 문구 (상단 1회 표시)

## 5. 행동 (Actions)

### Primary Action (핵심 행동)

- 버튼 라벨: "이 초안으로 시작하기"
- 행동 설명: 현재 초안을 수락하고 편집기 화면으로 이동
- 행동 후 결과: 편집기 화면이 열리며 초안 본문과 제목이 편집 가능한 상태로 로드됨

### Secondary Actions (보조 행동)

| 행동          | 트리거                       | 결과                                                         |
| ------------- | ---------------------------- | ------------------------------------------------------------ |
| 다시 생성하기 | 보조 버튼 (아웃라인 스타일)  | 수정 지시사항 입력 필드 표시 후 AI 재생성                    |
| 초안 복사하기 | 텍스트 링크 또는 아이콘      | 초안 본문을 클립보드에 복사, 토스트로 "복사됨" 표시          |
| 뒤로 가기     | 상단 ← 아이콘               | 주제 입력 화면으로 복귀 (입력값 유지)                        |

## 6. 입력과 검증 (Input & Validation)

| 입력 필드      | 타입          | 필수                        | 제약 조건  | 검증 실패 메시지             |
| -------------- | ------------- | --------------------------- | ---------- | ---------------------------- |
| 수정 지시사항  | 텍스트 영역   | 아니오 (재생성 시에만 노출) | 최대 200자 | "200자 이내로 입력해 주세요" |

## 7. 내비게이션과 흐름 (Navigation & Flow)

### 화면 상태 (정상 경로)

| 상태        | 조건                  | 화면 반응                                                                |
| ----------- | --------------------- | ------------------------------------------------------------------------ |
| 로딩 중     | AI 생성 API 호출 중   | Spinner + "초안을 만들고 있어요..." 문구                                 |
| 정상 표시   | API 응답 성공         | 초안 본문/제목 표시, Primary Action 버튼 활성화                          |
| 재생성 중   | 재생성 요청 후 대기   | 기존 초안 희미하게 + Spinner + "다시 만들고 있어요..." 문구              |
| 데이터 없음 | 생성 결과가 비어있음  | EmptyState 패턴 — "초안을 생성하지 못했어요. 다시 시도해 주세요." 표시   |

### 화면 흐름 & 컨텍스트

- 진입: ← [주제 입력 화면] (전달받는 데이터: 주제, 톤, 길이 옵션)
- 이탈: → [편집기 화면] (전달하는 데이터: 초안 본문, 제목)
- 뒤로 가기: ← [주제 입력 화면] (입력값 유지)

### AI 분기 조건

| AI 응답 상태 | 화면 처리                                               | 이동 경로              |
| ------------ | ------------------------------------------------------- | ---------------------- |
| 정상 완료    | 초안 본문/제목 표시                                     | 현재 화면 유지         |
| 생성 실패    | Alert(error) — "초안 생성에 실패했어요."                | 현재 화면 유지         |
| 콘텐츠 거부  | Alert(warning) — "요청하신 주제로는 글을 만들 수 없어요." | 현재 화면 유지       |
| 타임아웃     | Alert(error) — "시간이 초과되었어요. 다시 시도해 주세요." | 현재 화면 유지       |

### 이탈 가드

- 경고 필요 여부: 아니오
- 이유: 초안은 언제든 재생성 가능하므로 데이터 손실 우려 없음

## 8. 예외와 오류 처리 (Edge Cases & Error Handling)

| 상황                      | 화면 반응                                       | 안내 메시지                                                |
| ------------------------- | ----------------------------------------------- | ---------------------------------------------------------- |
| 데이터 없음 (빈 응답)    | EmptyState 패턴 + "다시 생성하기" 버튼          | "초안을 생성하지 못했어요. 다시 시도해 주세요."           |
| 네트워크 오류             | Alert(error) + "재시도" 버튼                    | "네트워크 연결을 확인해 주세요."                          |
| AI 서비스 장애            | Alert(error) + "잠시 후 다시 시도" 버튼         | "AI 서비스에 일시적인 문제가 있어요. 잠시 후 다시 시도해 주세요." |
| 콘텐츠 정책 위반          | Alert(warning) + 주제 수정 유도                 | "요청하신 주제로는 글을 만들 수 없어요. 다른 주제를 시도해 주세요." |
| 재생성 횟수 초과 (5회)   | "다시 생성하기" 버튼 비활성화                   | "재생성은 5회까지 가능해요. 현재 초안을 편집해 보세요."   |
| 초안 길이 초과 (3,000자) | 본문 잘림 + 하단 안내                           | "초안이 길어 일부만 표시돼요. 편집기에서 전문을 확인하세요." |

## 9. 감정 방향 (Emotional Direction)

- 키워드: 성취감, 기대감, 안심
- UX 라이팅 원칙: "AI가 해냈어요!" 톤. 사용자가 결과물을 보고 만족감을 느끼도록, 긍정적이고 격려하는 어조 사용
- 하지 말아야 할 톤: 사무적이거나 기계적인 어조. "생성이 완료되었습니다" 대신 "초안이 준비됐어요!"

## 10. 제약과 전제 (Constraints & Assumptions)

### 기술 제약

- AI 생성 API 응답 시간 최대 30초, 초과 시 타임아웃 처리
- 초안 본문 최대 3,000자 제한 (API 레벨)
- 재생성 시 이전 초안은 보관하지 않음 (새 초안으로 교체)

### 비즈니스 규칙

- 재생성 횟수 사용자당 1일 최대 5회
- 콘텐츠 정책 위반 시 생성 차단 (API 레벨에서 필터링)
- 초안 수락 전에는 편집기 접근 불가

### 전제 조건

- 사용자가 이전 화면에서 주제, 톤, 길이를 모두 선택한 상태
- AI 생성 API가 정상 동작 중
- 사용자가 로그인된 상태
